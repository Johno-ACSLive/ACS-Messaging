# Contributing

If you'd like to contribute, that would be fantastic as we could definitely use your help!


## Code of conduct

Any contribution or participation on this project is expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).


## Getting Started

If you're looking to start contributing you can [Create an Issue](https://github.com/Johno-ACSLive/ACS-Messaging/issues), [Create a Pull Request](https://github.com/Johno-ACSLive/ACS-Messaging/pulls), [Create a Discussion](https://github.com/Johno-ACSLive/ACS-Messaging/discussions) or [Contact Us](https://www.acslive.com.au/contact/).

For bug fixes or small changes please create a pull request. For anything else, please contact us so we can discuss.

For large changes or architectural changes please create a discussion.

## Reporting Issues

- **Bug Reports**: [Create an Issue](https://github.com/Johno-ACSLive/ACS-Messaging/issues) describing the problem, steps to reproduce, and expected behavior
- **Feature Requests**: [Create an Issue](https://github.com/Johno-ACSLive/ACS-Messaging/issues) describing the proposed feature and use case
- **Security Vulnerabilities**: Please report privately via [Contact Us](https://www.acslive.com.au/contact/) - do not create public issues


## Getting Help

For questions and community support:
- Review the sample projects ([Sample Client](https://github.com/Johno-ACSLive/ACS-Messaging/tree/master/Sample%20Client) and [Sample Server](https://github.com/Johno-ACSLive/ACS-Messaging/tree/master/Sample%20Server)) in the repository
- [Create a Discussion](https://github.com/Johno-ACSLive/ACS-Messaging/discussions) on GitHub
- [Contact Us](https://www.acslive.com.au/contact/) for commercial support inquiries
